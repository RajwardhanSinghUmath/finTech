**Prerequisites**



- Node.js 18+ (LTS recommended)

- Git

- One of: `npm`, `yarn`, `pnpm`, or `bun`



**1) Clone the repo**



```bash

git clone <your-repo-url>

cd <your-repo-name>

```



**2) Install dependencies**



Use whichever package manager you prefer:



```bash

npm install

# or

yarn install

# or

pnpm install

# or

bun install

```



**3) Configure environment variables**



Create a `.env` file in the project root and add the keys below. Replace the example values with your real secrets.



```env

NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=

CLERK_SECRET_KEY=

GROQ_API_KEY=

NEXT_PUBLIC_SUPABASE_URL=

NEXT_PUBLIC_SUPABASE_ANON_KEY=

```



How to obtain these keys:



- **Clerk**: Sign in at https://clerk.com → create an application → Dashboard → API Keys (Publishable & Secret).

- **Groq**: Sign in at https://app.groq.ai or the Groq Cloud Console → API Keys → Create API Key.

- **Supabase**: Sign in at https://supabase.com → create/select project → Project Settings → API → Project URL & anon key.



Keep `.env` out of version control (it's already in `.gitignore` by default).

**4) Define Database Schema**

You'll need a table in Supabase to store the eye-tracking sessions. You can run the following SQL in the Supabase [SQL Editor](https://supabase.com/dashboard/project/_/sql).

```sql
-- Create the table
create table eye_tracking_sessions (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  duration numeric,              -- Duration of the session in seconds
  converted boolean,             -- Whether the user completed a purchase
  confusion_events jsonb,        -- Array of confusion events detected
  gaze_points jsonb,             -- Array of gaze coordinates
  help_shown boolean             -- Whether the help modal was shown
);

-- (Optional) Enable Row Level Security if needed, or leave public for testing
alter table eye_tracking_sessions enable row level security;

create policy "Enable insert for all users" 
on eye_tracking_sessions for insert 
with check (true);

create policy "Enable read for all users" 
on eye_tracking_sessions for select 
using (true);
```

**5) Populate with Dummy Data (Optional)**

To test the admin panel without running a full session, you can insert some dummy data:

```sql
insert into eye_tracking_sessions 
(duration, converted, confusion_events, gaze_points, help_shown)
values
(
  120.5, 
  true, 
  '[{"timestamp": 5000, "reason": "rapid_movement"}]', 
  '[{"x": 100, "y": 200}, {"x": 150, "y": 250}]', 
  false
),
(
  45.0, 
  false, 
  '[]', 
  '[{"x": 500, "y": 500}]', 
  true
);
```



**6) Run the dev server**



```bash

npm run dev

# or

yarn dev

# or

pnpm dev

# or

bun dev

```



Open http://localhost:3000 in your browser.



You can edit the app starting at [app/page.js](app/page.js). Changes reload automatically.



**Build / Production**



```bash

npm run build

npm start

```



**Troubleshooting**



- If you see auth or API errors, confirm your keys are set in `.env.local` and that you restarted the dev server after changing the file.

- If installs fail, delete `node_modules` and lockfile (`package-lock.json` / `yarn.lock` / `pnpm-lock.yaml`) then reinstall.

- Port conflicts: set `PORT` env var before starting, e.g. `PORT=3001 npm run dev` (Windows PowerShell: `$env:PORT=3001; npm run dev`).



**Deploying**



On Vercel, set the same environment variables in the project settings (do not upload `.env.local`).
